Ok here is the high level
We are building a Home Gym Weight Lifting Tracking application
Built on Python
Built with Flask
Built with Gunicorn
Built with CoreUI Free / Bootstrap 5
Built with NGINX
Built with SQLite
Very Mobile Friendly

Primary Components:
- DB of all Possible Lifts/Activities in Home Gyms
- DB of all Possible Lifts/Activities in "MY" Home Gym
- DB of Lifting/Activity Programs
- DB of Historical Tracking of all lifts/Activities
- DB for Users and Profiles
- UI for adding Home Gyms lifts/Activities - and assigning them to "My" Gym, types of progressive changes to lifts
- UI for building a Lifting/Activity Program - assigning a "MY" Gym, lifts from that gym, days, routines, sets, reps, etc
- UI for reporting on per lifts status over time
- UI for calendar scheduling, and shifting program and lifting days around
- UI for at the Gym, working through and updating the program as you go
- Ui for tracking personal goals
- Multi User - Multi "My" Gym - or Shared Gyms (admin user to manage users)
